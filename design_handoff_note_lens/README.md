# 핸드오프: 노트 렌즈 (G드라이브 마크다운 시각화 도구)

> 이 문서 하나로 이 대화에 참여하지 않은 개발자/AI도 전체 기능을 구현할 수 있도록 작성했습니다.
> VS Code의 Claude(Claude Code)에게 이 폴더를 통째로 열어주고 "README를 읽고 구현 계획을 세워줘"라고 시작하세요.

## 1. Overview (개요)

**노트 렌즈**는 Google Drive에 마크다운(.md)으로 정리된 메모(프로젝트 진행상황, 회의·의사결정 기록, AI 룰/스킬 설정, 할 일, 작업 로그)를 **사람이 한눈에 이해하도록 시각화**하는 개인용 도구다. Obsidian과 유사한 볼트 구조를 가지되, 글만 보는 것보다 **글 + 그림(그래프·다이어그램·마인드맵)**으로 보면 구조 파악이 쉽다는 전제에서 출발한다.

핵심 요구사항:
1. **실시간 반영** — G드라이브의 md 변경이 자동으로 화면에 반영됨
2. **다중 시각화** — 같은 메모를 노드 그래프 / 카드 대시보드 / 마인드맵 / 자동 플로우차트 / 리치 문서로 전환해 보기
3. **인앱 편집(WYSIWYG)** — 화면 안에서 바로 편집하고 G드라이브에 다시 저장

주 사용자: **본인 1인 정리용** (멀티유저 협업/권한은 범위 밖).

## 2. About the Design Files (디자인 파일에 대하여)

이 번들의 `노트 렌즈.dc.html`은 **HTML로 만든 디자인 레퍼런스(프로토타입)** 다. 의도한 외형과 동작을 보여주는 목업이며, **그대로 프로덕션에 복사해 쓰는 코드가 아니다.**

**할 일:** 이 HTML 디자인을 대상 코드베이스의 환경에서 **재구현**한다.
- 기존 코드베이스가 있으면 그 프레임워크/패턴/라이브러리(React, Vue 등)를 따른다.
- 환경이 아직 없다면(신규 프로젝트), 아래 "권장 스택"을 참고해 적절한 프레임워크를 골라 구현한다.

데이터는 현재 프로토타입에서 **목업(하드코딩)**으로 들어있다 — 실제로는 **Google Drive의 md 파일에서 읽어와야** 한다.

---

## 3. Fidelity (완성도)

**High-fidelity (hifi).** 색상·타이포그래피·간격·인터랙션이 최종안에 가깝다. UI는 아래 디자인 토큰을 사용해 픽셀에 가깝게 재현하면 된다. 단, SVG 시각화(그래프/마인드맵/다이어그램)의 **노드 좌표는 예시**이며, 실제로는 자동 레이아웃 알고리즘으로 계산해야 한다(아래 5.2/5.4/5.5 참고).

---

## 4. 권장 스택 (신규 프로젝트일 경우)

확정은 아니며 코드베이스 상황에 맞게 조정.
- **프론트엔드:** React + TypeScript + Vite
- **마크다운 파싱:** `unified` + `remark-parse` + `remark-gfm` → mdast(AST). AST를 순회해 블록 렌더링과 다이어그램/마인드맵 데이터를 모두 생성.
- **그래프 레이아웃:** `d3-force` (force-directed). 소규모면 정적 레이아웃도 가능.
- **마인드맵/플로우차트:** 직접 SVG 렌더 또는 `reactflow`/`elkjs`(계층 레이아웃) 활용.
- **WYSIWYG 편집:** `Tiptap`(ProseMirror 기반) 또는 `milkdown`. 마크다운 ↔ 리치 양방향 변환이 핵심.
- **Google Drive 연동:** Drive API v3. 인증은 OAuth 2.0. **실시간**은 Drive `changes.watch`(push notification/webhook) 또는 `changes.list` 폴링. (프로토타입의 의사결정 기록상 **Webhook 채택**이 결론.)

## 5. Screens / Views (화면별 상세)

전역 레이아웃(데스크톱 우선): 세로 column.
- **상단바(54px 고정):** 좌측 로고+제목 "노트 렌즈" + `G Drive · vault` 칩 / 중앙 검색창(폭 340px, max 42vw) / 우측 **동기화 상태 인디케이터**.
- **본문 row(나머지 높이):** `사이드바(252px 고정) | 메인(flex) | 우측 패널(286px 고정)`.
- 메인 = 상단 뷰 탭바(50px) + 스크롤 콘텐츠.

### 5.0 동기화 인디케이터 (상단바 우측)
- 두 상태: **동기화 중** = 12~13px 링(border 2px, border-top 투명)이 `spin .8s linear infinite`로 회전, 색 `#c4663d`, 라벨 "동기화 중…". **유휴** = `#7da06a` 채워진 점, 라벨 "방금 동기화됨" / "N분 전 동기화됨".
- 프로토타입은 11초마다 1.4초간 "동기화 중"을 시뮬레이션. **실제로는 Drive change 이벤트 수신 시 표시.**

### 5.1 사이드바 — 볼트 트리
- 헤더 "볼트"(모노폰트, 11px, letter-spacing .08em, 대문자 톤, `#a3957a`).
- 폴더 그룹: 프로젝트 / 회의·의사결정 / AI 설정 / 할 일 / 기록. 각 폴더는 8×8px 라운드 사각 색칩(유형색) + 이름(12.5px, 600, `#5b5346`) + 우측 파일 수(모노 10.5px).
- 파일 항목: 5px 원형 점(유형색) + 제목(13px). 선택 시 배경 `#eadfc6`, 글자 `#2c2620`/600. hover 시 배경 `#eadfc6`.
- **실제 구현:** 폴더/파일 = G드라이브 폴더 구조에서 자동 생성. 유형은 폴더명 또는 frontmatter(`type:`)로 매핑.

### 5.2 그래프 뷰 (노드 그래프)
- 제목 "연결 그래프" + 설명 "메모 사이의 [[링크]] 관계. 노드를 누르면 문서가 열립니다."
- SVG: 노드 = 원(프로젝트 r=21로 큼, 나머지 r=14), 채움=유형색, 테두리 `#f7f1e3` 3px. 선택 노드는 뒤에 반투명 헤일로(r+7, opacity .16). 노드 아래 라벨(12.5px). 엣지 = 직선, 평상시 `#cdbfa6`/1.4px, 선택 노드에 연결된 엣지는 `#c4663d`/2px 강조.
- 노드 클릭 → 해당 문서 뷰로 이동.
- **엣지 정의:** md 본문의 `[[위키링크]]` 또는 마크다운 링크에서 파일 간 참조를 추출. **좌표는 force-directed로 자동 계산**(프로토타입 좌표는 버릴 것).

### 5.3 대시보드 (카드)
- 제목 "볼트 대시보드". `grid`, `repeat(auto-fill, minmax(280px,1fr))`, gap 16px.
- 카드: 배경 `#fffdf7`, 1px `#e7dcc4`, radius 12px, padding 18px, min-height 158px. 상단 유형 색점+모노 라벨(대문자), 제목(Spectral 18px/600), 요약(13px `#7a705e`), 하단 메타(모노 10.5px): `↻ 수정시각`, `🔗 링크수`. hover 시 살짝 떠오름 + 그림자.
- 카드 클릭 → 문서 뷰.
- **요약:** frontmatter `summary:` 또는 본문 첫 문단/AI 요약.

### 5.4 마인드맵
- 제목 "마인드맵 · {문서제목}". 현재 선택 문서의 **머리글(h1→h2→…) 계층**을 좌→우 트리로 펼침.
- 루트 노드(유형색 채움) → 분기 노드(연한 배경 `#fbf2e0`/테두리 `#e0b894`) → 리프(흰 배경). 연결은 bezier 곡선(`#cdbfa6`/`#ddd0b4`).
- **실제:** mdast의 heading depth로 트리 구성. 좌표/곡선은 계층 레이아웃(elk 등)으로 자동 계산.

### 5.5 자동 다이어그램 (플로우차트)
- 제목 "자동 다이어그램". 의사결정·절차가 담긴 글을 플로우차트로 변환.
- 노드 박스 + 화살표(끝에 ▷ 모양). 시작/결과 노드는 pill(둥근), 결정 노드는 강조색(`#6d7e84` 채움/흰 글자). 분기는 좌우로 갈라짐.
- **실제:** 본문의 순서 있는 리스트/"옵션/결정/실행" 같은 구조나 mermaid 코드블록을 파싱해 노드·엣지 생성. 좌표는 자동 레이아웃.

### 5.6 문서 뷰 (리치 렌더 + 편집)
- 폭 760px 중앙 정렬, padding 34/40/80px.
- 상단 메타: 유형 색점 + 모노 유형 라벨(유형색, 대문자) + `· 수정시각`.
- 블록 렌더:
  - **h1:** Spectral 32px/700, letter-spacing -.015em, line-height 1.15
  - **h2:** Spectral 21px/600, margin-top 26px
  - **p:** 15.5px/line-height 1.7, `#3a342b`
  - **list:** 5px 점(`#c4663d`) + 항목(15px/1.6)
  - **todo:** 19px 체크박스(완료 시 `#7da06a` 채움+흰 ✓, 미완료 시 `#cdbfa0` 테두리), 완료 항목은 취소선+`#a3957a`
  - **callout:** 배경 `#fbf2e0`/테두리 `#ecdcbf`/radius 10px, 좌측 24px ★ 배지(`#c4663d`), 제목+본문
  - **quote:** 좌측 3px `#c4663d` 보더, Spectral italic 17px
  - **img:** 200px 높이 대각 줄무늬 플레이스홀더, "[ 이미지 ]" + 캡션. (실제 G드라이브 첨부/임베드 이미지로 대체)
  - **divider:** 1px `#e2d6bd`
- 하단 "연결된 메모" 칩 목록 → 클릭 시 해당 문서로.
- **편집 모드:** 우상단 "편집" 버튼(`#fbf2e0` 바탕/`#c4663d` 글자) 토글 → "완료"(`#c4663d` 바탕/흰 글자).
  - 편집 중: 상단 sticky 툴바 등장 — "편집 중" 라벨, 포맷 버튼(B / I / H / " / • / ✓ / 🖼), 우측 "● 자동 저장됨" 표시.
  - 편집 가능한 블록(h1/h2/p/list 항목/todo 텍스트/callout 본문)은 `contentEditable`, focus 시 배경 `#fbf3df`.
  - **실제:** Tiptap 등으로 마크다운 양방향 편집 → 변경분을 Drive 파일에 저장(디바운스 자동저장).

### 5.7 우측 패널
- **문서 뷰일 때:** 목차(머리글 리스트, h2 들여쓰기) / 백링크(이 문서를 링크한 다른 메모 카드, 클릭 이동, 없으면 "아직 연결된 메모 없음") / 속성(유형·수정시각·#태그 칩).
- **그래프·대시보드·마인드맵·다이어그램일 때:** 범례(유형별 색) + 안내 문구.

## 6. Interactions & Behavior

- **뷰 전환:** 상단 탭(대시보드/그래프/문서/마인드맵/다이어그램) 클릭 → `view` 상태 변경. 활성 탭은 `#c4663d` 배경/흰 글자.
- **문서 열기:** 사이드바 항목 / 카드 / 그래프 노드 / 백링크 / 연결칩 클릭 → `view='doc'`, `selectedId=대상`.
- **편집 토글:** 문서 뷰에서만 노출.
- **동기화 표시:** Drive 변경 수신 시 "동기화 중…" 회전 → 완료 시 "방금 동기화됨"으로 복귀, 변경된 메모는 즉시 재렌더.
- **애니메이션:** 동기화 링 `spin .8s linear infinite`. 카드 hover `translateY(-2px)` + 그림자. (과한 전환 없음 — 차분한 종이 톤 유지.)

## 7. State Management

- `view`: `'dashboard' | 'graph' | 'doc' | 'mindmap' | 'diagram'`
- `selectedId`: 현재 선택된 파일 id
- `editing`: boolean (문서 편집 모드)
- `syncing`: boolean / `lastSyncedAt`: timestamp (동기화 인디케이터)
- **데이터(실제 구현 추가):** `files`(Drive에서 로드한 파일 메타+파싱된 블록), 파일 간 링크 그래프, frontmatter(type/tags/summary). Drive change 이벤트 → 해당 파일 재파싱 → 상태 갱신.

데이터 모델(파일 1개):
```
{ id, folder, type, title, updatedAt, tags: string[],
  links: string[],        // 이 문서가 참조하는 파일 id들 (위키링크/링크에서 추출)
  summary: string,
  blocks: Block[] }        // mdast에서 변환한 렌더 블록
Block = { type: 'h1'|'h2'|'p'|'list'|'todo'|'callout'|'quote'|'img'|'divider', ... }
```

## 11. 구현 순서 제안 (선택)

1. Drive OAuth + 폴더/파일 목록 로드 → 사이드바
2. md 파싱(remark) → 문서 뷰 블록 렌더(읽기 전용)
3. 링크 추출 → 그래프 뷰(d3-force) + 백링크/우측 패널
4. 대시보드 카드
5. WYSIWYG 편집(Tiptap) + Drive 저장
6. 실시간 동기화(changes.watch) + 인디케이터
7. 마인드맵·자동 다이어그램(heading/구조 기반 자동 레이아웃)
